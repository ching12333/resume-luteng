# 简历编辑器 - 编辑器与网站分离说明

## 1. 分离概述

原项目将所有代码（HTML结构、CSS样式、JavaScript逻辑、简历数据）全部集中在一个 `index.html` 文件中。现已完成拆分，将**编辑器**（负责交互逻辑）与**网站数据**（简历内容）彻底分离。

## 2. 新的文件结构

| 文件 | 类型 | 作用 | 更新频率 |
|------|------|------|----------|
| `index.html` | 壳文件 | HTML 结构骨架，引用其他文件 | 极少更新 |
| `editor.css` | 编辑器样式 | 所有 CSS 样式 | 随编辑器版本更新 |
| `editor.js` | 编辑器逻辑 | 所有交互逻辑、GitHub API 保存等 | 随编辑器版本更新 |
| `resume-data.js` | 网站数据 | 简历内容（INIT_ELEMENTS 数组） | 每次编辑保存时更新 |
| `assets/` | 资源目录 | 图片资源（照片、二维码等） | 按需更新 |

## 3. 分离带来的好处

**更新编辑器不影响简历数据：** 当需要升级编辑器功能（如新增工具、修复 bug、改进交互）时，只需替换 `editor.js` 和/或 `editor.css`，`resume-data.js` 中的简历内容完全不受影响。

**保存简历不修改编辑器代码：** 用户在网页上编辑简历后点击「保存到网址」，系统只会更新 `resume-data.js` 这一个文件，编辑器代码保持不变。

**版本管理更清晰：** 在 Git 历史中可以清楚区分哪些提交是编辑器功能更新，哪些是简历内容修改。

## 4. 如何更新编辑器

当您开发了新版本的编辑器时，只需要：

1. 修改 `editor.js`（逻辑变更）和/或 `editor.css`（样式变更）
2. 提交并推送到 GitHub
3. 简历数据 `resume-data.js` 完全不需要改动
4. GitHub Pages 自动部署后，新版编辑器立即作用于现有的简历数据

## 5. 如何编辑简历内容

操作方式与之前完全相同：

1. 打开 `https://ching12333.github.io/resume-luteng/`
2. 在页面上拖拽、编辑文字、调整样式
3. 点击「☁ 保存到网址」按钮
4. 系统自动将最新简历数据推送到 GitHub 的 `resume-data.js` 文件
5. 约 1 分钟后刷新页面即可看到更新

## 6. 技术细节

**加载顺序：** `index.html` 先加载 `editor.css`（样式），然后加载 `resume-data.js`（数据），最后加载 `editor.js`（逻辑）。`editor.js` 的 `init()` 函数会读取 `INIT_ELEMENTS` 变量来渲染简历。

**保存机制变更：** 原来保存时会修改 `index.html` 中的 `INIT_ELEMENTS` 数据块。现在保存时只生成并推送 `resume-data.js` 文件，编辑器代码完全不被触碰。

**GitHub 配置：** Token、仓库信息等配置仍在 `editor.js` 的顶部，迁移账号时只需修改 `editor.js` 中的相关变量。
